# Java 14: New Features (17 March 2020)

- Pattern Matching for instanceof [JEP-305](https://openjdk.org/jeps/305)
- Packaging tool [JEP-343](https://openjdk.org/jeps/343)
- NUMA-Aware Memory Allocation for G1 [JEP-345](https://openjdk.org/jeps/345): In a multi-CPU system, Java's G1 Garbage Collector allocates memory near to the CPU that uses it.

- JFR Event Streaming [JEP-349](https://openjdk.org/jeps/354): Java Flight Recorder collects events that occur in the JVM when the Java application runs.

- Non-Volatile Mapped Byte Buffers [JEP-352](https://openjdk.org/jeps/352): Standard api to take advantage of Non-Volatile Memory storage.

- Helpful NullPointerExceptions [JEP-358](https://openjdk.org/jeps/358)
- Records (Preview) [JEP-359](https://openjdk.org/jeps/359)
- Switch Expressions [JEP-361](https://openjdk.org/jeps/361)
- Deprecate the Solaris and SPARC Ports [JEP-362](https://openjdk.org/jeps/362): Removes JVM supports for Solaris os and SPARC architecture

- Remove the Concurrent Mark Sweep (CMS) Garbage Collector [JEP-363](https://openjdk.org/jeps/363)
- ZGC on macOS [JEP-364](https://openjdk.org/jeps/364)
- ZGC on Windows [JEP-365](https://openjdk.org/jeps/365)
- Deprecate the ParallelScavenge + SerialOld GC Combination [JEP-366](https://openjdk.org/jeps/366)
- Remove the Pack200 Tools and API [JEP-367](https://openjdk.org/jeps/367)
- Text Blocks [JEP-368](https://openjdk.org/jeps/368)
- Foreign-Memory Access API [JEP-370](https://openjdk.org/jeps/370)
